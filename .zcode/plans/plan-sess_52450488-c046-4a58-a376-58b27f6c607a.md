## Goal
Properly configure React→Django API calls: route through a Vite dev proxy, centralize all backend access in a typed `src/api/` client, remove the dead WebSocket code path, wire the unused `/api/nodes/` and `/api/alerts/` endpoints into the app, and fix the affected tests/docs.

## Backend reality (no changes needed for functionality)
- WSGI-only Django + DRF. Endpoints: `/api/telemetry/latest/`, `/api/latest/`, `/api/nodes/`, `/api/alerts/`.
- `CORS_ALLOW_ALL_ORIGINS = True` already set — keep as-is (harmless with the proxy; useful if frontend is ever served from another origin in prod). The existing CORS test keeps passing.

## Frontend changes

### 1. `frontend/vite.config.ts` — add dev proxy
Add `server.proxy` so `/api` requests are forwarded to Django in dev (no CORS, no hardcoded `localhost:8000` in app code):
```ts
server: {
  port: 3000,
  open: false,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    },
  },
},
```

### 2. `frontend/src/vite-env.d.ts` — type the env var
```ts
/// <reference types="vite/client" />
interface ImportMetaEnv {
  readonly VITE_API_BASE_URL?: string;
}
interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```
`VITE_API_BASE_URL` defaults to `''` (relative `/api/...` → works via dev proxy in dev, reverse proxy in prod). Optional escape hatch for non-proxy deployments — no env file required for local dev.

### 3. NEW `frontend/src/api/client.ts` — centralized fetch wrapper
- Reads `VITE_API_BASE_URL` (default `''`), builds `API_BASE`.
- `request<T>(path)`: `fetch` with JSON parse, throws on non-2xx, unwraps the `{status, source, data}` envelope (`payload.data ?? payload`).
- Small typed functions: `getLatestTelemetry()`, `getNodes()`, `getAlerts()`.
- All paths are relative (`/api/telemetry/latest/`, etc.) → resolved through the proxy.

### 4. NEW `frontend/src/types/api.ts` — backend shapes + adapters
- `ApiTelemetry`, `ApiNode`, `ApiAlert` interfaces matching the DRF serializers.
- Adapter `mapAlertToAlertItem(alert: ApiAlert): AlertItem` (severity→level, node_id→sector, triggered_at→timestamp, resolved_at→status). This is needed because the backend `Alert` shape differs from the display `AlertItem` type.
- `ApiNode` stays as-is (it doesn't map to `StructuralNode` — no stress/vibration fields — so nodes get their own context field, see step 6).

### 5. `frontend/src/context/TelemetryContext.tsx` — rewrite the connection logic
- **Remove all WebSocket code** (`getWsUrl`, `wsUrl`, `connect()`, WS handlers, `isWsActive`, `VITE_WS_URL`). Backend has no WS server.
- Telemetry polling (3s, existing interval) now calls `getLatestTelemetry()` instead of raw `fetch(API_URL)`.
- Add `nodes` state (`ApiNode[]`) and expose it via context. New effect fetches `getNodes()` + `getAlerts()` on mount and refreshes every 10s; alerts are run through `mapAlertToAlertItem` and merged into existing `activeAlerts`.
- Keep `connectionStatus` / `connectionError` / `reconnect()` semantics intact (driven by fetch success/failure now) so `BackendErrorOverlay` still works unchanged.
- Add `nodes: ApiNode[]` to `TelemetryContextType`.

### 6. `frontend/src/components/views/InfrastructureView.tsx` — render nodes
Add a compact "REGISTERED SENSOR NODES" section that maps over the new `nodes` from context (id, name, type, status, location). Keeps the existing structural-nodes section. This makes the `/api/nodes/` endpoint genuinely consumed (not just fetched).

### 7. `frontend/src/setupTests.ts` — remove MockWebSocket
Delete the `MockWebSocket` class and the global assignment. No longer needed once WS code is gone.

### 8. Rewrite the 3 test files to use `fetch` mocks (not WS)
- `__tests__/context/TelemetryContext.test.tsx`: replace WS-driven tests with `vi.stubGlobal('fetch', …)` returning telemetry/node/alert payloads; assert `fetch` called with `/api/telemetry/latest/` and that data flows into context; simulate fetch rejection → `connectionStatus === 'error'`.
- `__tests__/components/common/BackendErrorOverlay.test.tsx`: trigger the overlay via a failing `fetch` (instead of `ws.triggerError`).
- `__tests__/views/OverviewDashboard.test.tsx`: push telemetry via the fetch mock instead of `ws.triggerMessage`.
- Remove `VITE_WS_URL` stubs and `MockWebSocket` imports everywhere.

### 9. `frontend/.env.example` — update
Remove `VITE_WS_URL`. Document the optional `VITE_API_BASE_URL` override and the dev-proxy default:
```
# Optional: override the API base for production (default is relative "/api", proxied in dev).
# VITE_API_BASE_URL=https://api.example.com
```
